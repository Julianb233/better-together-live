<task type="auto">
  <name>Task 1: Fix password reset endpoints and add auth callback route</name>
  <files>src/api/supabase-auth.ts</files>
  <action>
**Fix the reset-password endpoint:**

The current reset-password endpoint uses `createServerClient(env)` (renamed to `createAnonClient` in 02-01) which creates a plain client with no session. But `supabase.auth.updateUser()` requires an authenticated session -- the user must be logged in via the recovery token.

Update the reset-password endpoint to:
1. Accept `{ newPassword, accessToken, refreshToken }` in the body (the frontend will pass these from the URL hash fragment after the callback)
2. Use `createAnonClient(env)` to create a client
3. Call `supabase.auth.setSession({ access_token: accessToken, refresh_token: refreshToken })` to establish the recovery session
4. Then call `supabase.auth.updateUser({ password: newPassword })`
5. On success, set auth cookies via `setSupabaseAuthCookies` so the user stays logged in

**Add GET /callback route:**

Add a GET endpoint at `/callback` on the supabaseAuth router. This handles the redirect from the password reset email link.

The Supabase email link redirects to the `redirectTo` URL with query params: `?code=...` (PKCE) or hash fragment `#access_token=...&refresh_token=...&type=recovery`.

For the PKCE flow (which @supabase/ssr uses):
1. Read `code` from query params: `c.req.query('code')`
2. If code exists, use `createAnonClient(env)` and call `supabase.auth.exchangeCodeForSession(code)`
3. If the session type is `recovery`, redirect to `/auth/reset-password` page with access_token and refresh_token as query params (NOT hash -- server-rendered pages can't read hash fragments)
4. For other types (signup confirmation), redirect to `/login?verified=true`
5. On error, redirect to `/login?error=invalid_link`

**Update forgot-password redirectTo:**

Change the `redirectTo` in the forgot-password endpoint from:
```
`${c.req.url.split('/api')[0]}/auth/reset-password`
```
to:
```
`${c.req.url.split('/api')[0]}/api/auth/callback`
```

This ensures the email link goes to the callback endpoint first (to exchange the code), not directly to the reset page.

**Do NOT change** the signup, login, logout, refresh, me, oauth, or update-profile endpoints.
