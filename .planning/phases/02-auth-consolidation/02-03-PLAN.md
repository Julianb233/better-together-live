<task type="auto">
  <name>Task 1: Migrate API files from custom auth to Supabase middleware</name>
  <files>src/api/users.ts, src/api/social.ts, src/api/discovery.ts</files>
  <action>
**users.ts:**
- Remove `import { extractToken, verifyToken } from './auth'`
- The route handler currently calls `extractToken(c, 'access')` then `verifyToken(accessToken, c.env, 'access')` to get `payload.userId`
- Replace ALL such patterns with `const userId = c.get('userId')` -- the global Supabase middleware already provides this
- If userId is null/undefined, return 401 (though this shouldn't happen since global middleware already guards)
- Remove any other references to the custom auth module

**social.ts:**
- Remove `import { requireAuth } from './auth'`
- Remove ALL inline `requireAuth` middleware arguments from route handlers. Currently every route has `socialApi.post('/...', requireAuth, async (c) => {...})` -- change to `socialApi.post('/...', async (c) => {...})`
- The global Supabase requireAuth middleware already protects all /api/* routes, so per-route auth is redundant
- Where handlers need the userId, use `c.get('userId')` instead of any custom auth extraction
- Check if handlers currently access `c.get('userId')` already (set by the inline custom requireAuth) -- if so, no change needed inside the handler body since the global Supabase middleware sets the same context key

**discovery.ts:**
- Remove `import { verifyToken, extractToken } from './auth'`
- Remove the `getAuthUserId` helper function that calls extractToken/verifyToken
- Replace all calls to `getAuthUserId(c)` with `c.get('userId')`
- The global middleware already validates auth and sets userId

**For all three files:** After removing auth.ts imports, verify no other imports from './auth' remain. The only auth-related imports should be from '../lib/supabase/middleware' or '../lib/supabase' if needed (but they shouldn't be, since global middleware handles everything).
  </action>
  <verify>
Run `npm run build` -- must pass. Verify:
- `grep -r "from './auth'" src/api/users.ts src/api/social.ts src/api/discovery.ts` returns NO matches
- `grep -r "extractToken\|verifyToken" src/api/users.ts src/api/discovery.ts` returns NO matches
- `grep -r "requireAuth" src/api/social.ts` returns NO matches (no inline middleware)
- `grep -r "c.get('userId')" src/api/users.ts src/api/social.ts src/api/discovery.ts` returns matches (they use global middleware context)
  </verify>
  <done>
users.ts, social.ts, and discovery.ts use c.get('userId') from global Supabase middleware. No imports from custom auth module remain.
  </done>

**Delete custom JWT auth files:**
- Delete `src/api/auth.ts` (custom JWT functions: generateAccessToken, verifyToken, hashPassword, extractToken, requireAuth, checkRateLimit)
- Delete `src/api/auth-routes.ts` (custom JWT route handlers: /register, /login, /logout, /forgot-password, /reset-password, /refresh)

**Update index.tsx route mounts:**
- Remove `import authRoutes from './api/auth-routes'`
- Remove `app.route('/api/auth', authRoutes)`
- Change `app.route('/api/auth/supabase', supabaseAuth)` to `app.route('/api/auth', supabaseAuth)`
- Update the public route whitelist in the `except()` call: remove `/api/auth/supabase/*` since supabaseAuth is now mounted at `/api/auth` (which is already in the whitelist as `/api/auth/*`)
- Verify no other imports reference auth.ts or auth-routes.ts

**Update login-system.ts:**
- The login form's handleLogin currently POSTs to `/api/auth/login` and stores `data.token` in localStorage
- Change the fetch URL to `/api/auth/login` (same path -- supabaseAuth is now mounted at /api/auth)
- Add `credentials: 'include'` to the fetch options (cookies need to be sent/received)
- Remove `localStorage.setItem('authToken', data.token)` -- session is in HTTP-only cookies
- On successful login, redirect to dashboard: `window.location.href = '/dashboard'`
- The register form's handleRegister currently POSTs to `/api/auth/register`
- Change to POST to `/api/auth/signup` (Supabase endpoint)
- Update the request body to match Supabase signup expectations: `{ email, password, name }` (check what the current form sends vs what supabase-auth.ts /signup expects)
- Add `credentials: 'include'` to fetch
- Remove any `localStorage.setItem('authToken', ...)` after registration
- On successful registration with session: redirect to dashboard
- On successful registration requiring email confirmation: show confirmation message

**Update login.ts:**
- Remove `localStorage.setItem('bt_authenticated', 'true')` and `localStorage.setItem('bt_user', ...)`
- Change auth checks to use cookie-based session: instead of checking localStorage, call `fetch('/api/auth/me', { credentials: 'include' })` to check if user is authenticated
- On successful login response, redirect to dashboard (cookies are set automatically by the server)
- Remove all `localStorage.getItem('bt_authenticated')` and `localStorage.getItem('bt_user')` references

**Update user-portal.ts:**
- Remove `localStorage.getItem('authToken')` usage for API calls
- Remove `localStorage.setItem('authToken', data.accessToken)` token refresh logic
- Remove `localStorage.removeItem('authToken')` on logout
- All fetch calls should use `credentials: 'include'` instead of `Authorization: Bearer ${token}` headers
- The `getAuthToken()` helper function should be removed or replaced with a no-op (cookies handle auth automatically)
- For the logout flow: POST to `/api/auth/logout` with `credentials: 'include'`, then redirect

**Update dashboard.ts:**
- Remove `localStorage.getItem('bt_authenticated') === 'true'` check
- Remove `localStorage.getItem('bt_user')` parsing
- Remove `localStorage.removeItem('bt_authenticated')` and `localStorage.removeItem('bt_user')`
- Replace auth check with: `fetch('/api/auth/me', { credentials: 'include' }).then(r => { if (!r.ok) window.location.href = '/login' })`
- For user data display, use the response from /api/auth/me

**Update intimacy-challenges.ts:**
- Remove all `localStorage.getItem('authToken')` references
- Remove `Authorization: Bearer ${token}` headers from fetch calls
- Add `credentials: 'include'` to all fetch calls that need auth
- Remove `localStorage.removeItem('authToken')` on auth failure
- On 401 response, redirect to login: `window.location.href = '/login'`

**Remove jose dependency:**
- Run `grep -r "jose" src/` to verify no remaining imports
- If none, run `npm uninstall jose` to remove from package.json

**Important notes:**
- All fetch calls to /api/* endpoints from page scripts should include `credentials: 'include'` so cookies are sent
- No Bearer token headers should remain in any page script
- No localStorage auth storage should remain in any page script
