  <name>Task 1: Install Stripe SDK and rewrite lib/stripe.ts as client factory + plan config</name>
  <files>package.json, src/lib/stripe.ts</files>
  <action>
    1. Run `npm install stripe` to add the official Stripe SDK.

    2. Rewrite `src/lib/stripe.ts` completely. The current file has a manual Web Crypto HMAC verification function -- this is no longer needed because the SDK provides `constructEventAsync`. Replace the entire file with:

    - `createStripeClient(secretKey: string): Stripe` -- creates a Stripe instance with `apiVersion: '2025-06-30.basil'` and `typescript: true`. Do NOT create a global singleton -- env vars are per-request in Vercel edge, so create per-request.

    - `STRIPE_PLANS` object -- single source of truth for plan configuration. Two plans:
      - `'try-it-out'`: name "Try It Out", interval "month", amount 3000 ($30), priceId from `STRIPE_PRICE_TRY_IT_OUT` env var (fallback to `'price_placeholder_tryitout'`)
      - `'better-together'`: name "Better Together", interval "year", amount 24000 ($240), priceId from `STRIPE_PRICE_BETTER_TOGETHER` env var (fallback to `'price_placeholder_bt'`)

    - Export `PlanId` type as `keyof typeof STRIPE_PLANS`

    NOTE: The priceId values reference env vars. Since this runs in edge/Hono where env is per-request, store the env var NAMES in the config and resolve them at call time. Use a helper `getPriceId(planId: PlanId, env: Record<string, string>): string` that reads from env at request time.

    3. Delete the old `verifyStripeSignature` and `timingSafeEqual` functions entirely -- the SDK handles this.
  </action>
  <verify>
    - `npm ls stripe` shows stripe package installed
    - `grep -r "verifyStripeSignature" src/` returns NO results (old function removed)
    - `grep "createStripeClient" src/lib/stripe.ts` confirms new factory exists
    - `grep "STRIPE_PLANS" src/lib/stripe.ts` confirms plan config exists
    - TypeScript compiles: `npx tsc --noEmit src/lib/stripe.ts` (or check via build)
  </verify>
  <done>Stripe SDK installed. lib/stripe.ts exports createStripeClient, STRIPE_PLANS, PlanId, and getPriceId. No manual HMAC code remains.</done>

<task type="auto">
  <name>Task 2: Rewrite payments.ts API to use Stripe SDK with full webhook lifecycle and portal</name>
  <files>src/api/payments.ts, src/lib/validation/schemas/payments.ts</files>
  <action>
    Rewrite `src/api/payments.ts` to use the Stripe SDK instead of raw fetch. Keep the same route structure but replace internals.

    **Delete entirely:**
    - `stripeRequest()` helper function
    - `SUBSCRIPTION_TIERS` object (replaced by STRIPE_PLANS from lib/stripe.ts)
    - `STRIPE_API_VERSION` constant
    - Import of `verifyStripeSignature`

    **Add imports:**
    - `import Stripe from 'stripe'`
    - `import { createStripeClient, STRIPE_PLANS, getPriceId, type PlanId } from '../lib/stripe'`

    **Rewrite POST /create-checkout-session:**
    - Accept `{ planId, userId, email }` (change from `tierId` to `planId` to match STRIPE_PLANS keys)
    - Validate planId against STRIPE_PLANS keys (server-side whitelist -- never trust client price)
    - Get priceId via `getPriceId(planId, c.env)`
    - First, check if user already has a `stripe_customer_id` in users table. If yes, use `customer` param. If no, use `customer_email` param (Stripe creates customer automatically, webhook will store the ID)
    - Use `stripe.checkout.sessions.create()` with:
      - `mode: 'subscription'`
      - `line_items: [{ price: priceId, quantity: 1 }]` (pre-created Price, NOT inline price_data)
      - `subscription_data.metadata: { userId, planId }`
      - `client_reference_id: userId`
      - `success_url` and `cancel_url` using relative paths from request origin
      - `allow_promotion_codes: true`
    - Return `{ sessionId, url }`

    **Rewrite POST /webhook:**
    - Use `stripe.webhooks.constructEventAsync(body, signature, webhookSecret)` for verification (NOT the old manual HMAC). Must use `c.req.text()` for raw body BEFORE any JSON parsing.
    - Handle FIVE event types (not just 3):
      1. `checkout.session.completed`: Upsert subscription record. Store `stripe_customer_id` on both the subscription AND the users table (so future checkouts can use `customer` instead of `customer_email`). Use `plan_id` (not `tier_id`) column.
      2. `invoice.paid`: Update subscription status to 'active', update `current_period_start` and `current_period_end` from invoice.
      3. `invoice.payment_failed`: Update subscription status to 'past_due'.
      4. `customer.subscription.updated`: Sync status, canceled_at from Stripe subscription object.
      5. `customer.subscription.deleted`: Set status to 'canceled'.
    - Implement idempotency: Before processing, check if event.id was already processed. Store processed event IDs by adding a simple check -- query subscriptions table for `updated_at` to see if this event's subscription data is already current. (Lightweight approach: just use upserts with appropriate conflict handling.)

    **Add POST /create-portal-session (NEW):**
    - Accept `{ userId }` (authenticated)
    - Look up `stripe_customer_id` from users table
    - If no customer ID, return 404
    - Create portal session via `stripe.billingPortal.sessions.create({ customer: customerId, return_url })`
    - Return `{ url }`

    **Keep GET /subscription-status** but update:
    - Reference STRIPE_PLANS instead of SUBSCRIPTION_TIERS
    - Use `plan_id` column (not `tier_id`)

    **Keep GET /tiers** but update:
    - Return plans from STRIPE_PLANS with formatted prices

    **Simplify POST /cancel-subscription:**
    - Use `stripe.subscriptions.update(subId, { cancel_at_period_end: true })` instead of raw fetch
    - (Users can also cancel via Customer Portal, so this is a convenience endpoint)

    **Keep POST /create-gift-checkout** but update to use SDK:
    - Use `stripe.checkout.sessions.create()` with `mode: 'payment'`

    **Update validation schemas** in `src/lib/validation/schemas/payments.ts`:
    - Change `tierId` to `planId` in createCheckoutSchema
    - Ensure planId validates against known plan IDs ('try-it-out', 'better-together')
    - Add schema for create-portal-session endpoint: `{ userId: z.string() }`

    IMPORTANT: Access env vars via `c.env` (Hono context), NOT `process.env`. This is an edge runtime.
  </action>
  <verify>

    - `grep "billingPortal" src/api/payments.ts` confirms portal endpoint
    - `grep "invoice.paid" src/api/payments.ts` confirms full lifecycle handling
    - `grep "customer.subscription.deleted" src/api/payments.ts` confirms deletion handling
    - `npm run build` succeeds (or `npx tsc --noEmit` passes)
  </verify>
  <done>
