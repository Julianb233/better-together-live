  <action>
    1. Install dependencies:
       ```bash
       npm install ai @ai-sdk/anthropic @ai-sdk/openai
       ```

    2. Create `src/lib/ai/prompts.ts`:
       - Export `COACH_SYSTEM_PROMPT` constant with the relationship coaching system prompt
       - Must include: warm/empathetic tone, actionable advice focus, follow-up questions
       - Must include guardrails: NO medical/legal/financial advice, crisis resource referral for abuse/danger, stay on relationship topics
       - Keep responses concise (2-3 paragraphs max)

    3. Create `src/lib/ai/conversation.ts`:
       - Export `getConversationHistory(supabase, userId, relationshipId, limit = 20)` -- fetches last N messages from ai_coach_messages table ordered by created_at ascending
       - Export `saveMessages(supabase, messages: Array<{user_id, relationship_id, role, content, model_used}>)` -- inserts one or more messages via supabase.from('ai_coach_messages').insert()
       - Both functions take SupabaseClient as first param (from context, not imported globally)
       - Throw on Supabase errors (caller handles)

    4. Create `migrations/0006_ai_coach_messages.sql`:
       - Table `ai_coach_messages`: id (UUID PK, gen_random_uuid()), user_id (UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE), relationship_id (UUID NOT NULL), role (TEXT NOT NULL CHECK IN ('user', 'assistant')), content (TEXT NOT NULL), model_used (TEXT nullable), created_at (TIMESTAMPTZ DEFAULT NOW())
       - Index on (user_id, relationship_id, created_at) for efficient history queries
       - Table `ai_coach_rate_limits`: user_id (UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE), window_start (TIMESTAMPTZ NOT NULL), request_count (INTEGER DEFAULT 1), PRIMARY KEY (user_id, window_start)
  </action>

    - `cat migrations/0006_ai_coach_messages.sql` contains CREATE TABLE ai_coach_messages

<task type="auto">
  <name>Task 2: Replace keyword-matching stub with Claude API integration</name>
  <files>src/api/ai-coach.ts</files>
  <action>
    Rewrite `src/api/ai-coach.ts` to use real Claude API via Vercel AI SDK:

    1. Import `generateText` from `ai`, `anthropic` from `@ai-sdk/anthropic`
    2. Import `requireAuth` from `../lib/supabase/middleware`
    3. Import `COACH_SYSTEM_PROMPT` from `../lib/ai/prompts`
    4. Import `getConversationHistory`, `saveMessages` from `../lib/ai/conversation`

    5. Apply `requireAuth()` middleware to all routes: `aiCoachApi.use('/*', requireAuth())`

    6. Rewrite POST `/ask` handler:
       - Parse body with existing askCoachSchema (message + relationship_id)
       - Get userId from `c.get('userId')` and supabase from `c.get('supabase')`
       - Call `getConversationHistory(supabase, userId, relationship_id)` to get last 20 messages
       - Call `generateText({ model: anthropic('claude-haiku-4-5'), system: COACH_SYSTEM_PROMPT, messages: [...history, { role: 'user', content: message }] })`
       - Call `saveMessages(supabase, [userMsg, assistantMsg])` to persist both messages (model_used: 'claude-haiku-4-5' for assistant, null for user)
       - Return `{ response: text, tier: 'complex', model: 'claude-haiku-4-5', timestamp }`
       - Wrap in try/catch: ZodError -> 400, LLM errors -> 500 with friendly message "I'm having trouble responding right now. Please try again in a moment."

    7. Update GET `/history/:relationship_id` handler:
       - Get userId and supabase from context
       - Call `getConversationHistory(supabase, userId, relationshipId, 50)` (higher limit for history view)
       - Return `{ messages }` array

    8. Delete the old `generateAIResponse` function entirely -- no keyword matching should remain.

    IMPORTANT: The Vercel AI SDK auto-reads ANTHROPIC_API_KEY from environment variables. Do NOT manually configure the API key. The env var must be set in Vercel dashboard (documented in user_setup).

    IMPORTANT: Use `anthropic('claude-haiku-4-5')` not `anthropic('claude-3-haiku')`. The research confirms haiku-4-5 as the target model.
  </action>
