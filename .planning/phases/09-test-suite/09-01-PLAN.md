  <action>
    Create `tests/unit/utils.test.ts` testing all PURE functions from src/utils.ts (functions that do NOT require Supabase/env). The pure functions are:
    - generateId() - returns UUID v4 format, unique values
    - daysBetween(date1, date2) - calculates absolute day difference
    - getCurrentDate() - returns YYYY-MM-DD format
    - getCurrentDateTime() - returns ISO format
    - isValidEmail(email) - validates email format
    - getPartnerId(relationship, currentUserId) - returns the other user's ID
    - calculateHealthScore(avgConn, avgSat, freq, goalRate) - returns 0-100 score

    Follow the research examples closely. Test cases to include:

    **generateId:**
    - Returns UUID v4 format (regex match)
    - Generates unique IDs (Set of 100 should have size 100)

    **daysBetween:**
    - Same date returns 0
    - Known date pair returns correct days (e.g., Jan 1 to Jan 10 = 9)
    - Works regardless of order (absolute difference)

    **getCurrentDate:**
    - Returns YYYY-MM-DD format (regex match)

    **getCurrentDateTime:**
    - Returns ISO format string (includes T and Z)

    **isValidEmail:**
    - Accepts: user@example.com, name+tag@domain.co
    - Rejects: 'notanemail', '@missing.com', '', 'spaces here@test.com'

    **getPartnerId:**
    - Returns user_2_id when currentUser is user_1_id
    - Returns user_1_id when currentUser is user_2_id

    **calculateHealthScore:**
    - Returns 0 for all-zero inputs
    - Returns value between 0 and 100 for typical inputs
    - Returns 100 for perfect inputs (10, 10, 10, 1)

    Do NOT test async functions (getUserByEmail, getUserById, etc.) -- those require Supabase mocking and belong in integration tests.

    Import types from fixtures if needed for getPartnerId test data.
