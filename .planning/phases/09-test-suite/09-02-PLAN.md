    **Test approach:** Import the supabase-auth sub-router, mount it on a minimal Hono app, use `app.request()` with MOCK_ENV as 3rd parameter. Mock `src/lib/supabase` and `src/lib/supabase/server` at module level via `vi.mock()`.

    **CRITICAL:** Mock must happen BEFORE importing the route module. Use `vi.mock()` at top level (hoisted by Vitest).

    ```typescript
    import { describe, it, expect, vi, beforeEach } from 'vitest'
    import { Hono } from 'hono'
    import { createMockSupabaseClient } from '../helpers/mock-supabase'
    import { MOCK_ENV } from '../helpers/mock-env'

    // Mock Supabase modules BEFORE importing routes
    const mockClient = createMockSupabaseClient()
    vi.mock('../../src/lib/supabase', () => ({
      createAnonClient: vi.fn(() => mockClient),
      createAdminClient: vi.fn(() => mockClient),
      setSupabaseAuthCookies: vi.fn(),
      clearSupabaseAuthCookies: vi.fn(),
    }))
    vi.mock('../../src/lib/supabase/server', () => ({
      createAnonClient: vi.fn(() => mockClient),
      createAdminClient: vi.fn(() => mockClient),
    }))
    ```

    Then import routes AFTER mocks:
    ```typescript
    const { default: supabaseAuth } = await import('../../src/api/supabase-auth')
    ```

    **Test suites to write:**

    1. **POST /api/auth/signup**
       - Returns 400 when email is missing
       - Returns 400 when password is missing
       - Returns 400 for invalid email format
       - Returns 200/201 on valid signup (mock signUp returns success)
       - Returns appropriate error when Supabase signUp fails (mock signUp returns error)

    2. **POST /api/auth/login**
       - Returns 400 when email is missing
       - Returns 400 when password is missing
       - Returns 200 on valid login (mock signInWithPassword returns session)
       - Returns 401 when credentials are wrong (mock signInWithPassword returns error)

    3. **POST /api/auth/logout**
       - Returns 200 on logout (mock signOut returns success)

    4. **POST /api/auth/forgot-password**
       - Returns 400 when email is missing
       - Returns 200 on valid email (mock resetPasswordForEmail returns success)

    For each test, use `app.request()` with:
    - Correct method and path
    - JSON body with `Content-Type: application/json` header
    - MOCK_ENV as 3rd parameter

    Use `beforeEach(() => { vi.clearAllMocks() })` to reset mock state between tests.

    **Important:** Read `src/api/supabase-auth.ts` fully before writing tests to understand exact route paths, expected request shapes, and response formats. The route uses zValidator which returns 400 with Zod errors for invalid input. Adapt test expectations to match actual route behavior.

    **Important:** If the route module uses middleware that checks auth (e.g., requireAuth), you may need to mock that too. Check the imports in supabase-auth.ts.
