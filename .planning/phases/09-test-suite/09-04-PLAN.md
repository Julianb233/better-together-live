  <action>
    Modify `.github/workflows/deploy.yml` to add test execution in the `lint-and-test` job.

    **Add these steps AFTER "Install dependencies" and BEFORE "Run type checking":**

    ```yaml
    - name: Run tests with coverage
      run: npx vitest run --coverage

    - name: Upload coverage report
      if: always()
      uses: actions/upload-artifact@v4
      with:
        name: coverage-report
        path: coverage/
        retention-days: 14
    ```

    **Why before type-checking:** Tests should fail fast. If tests fail, we don't need to wait for tsc. The existing type-check step has `|| echo "Type checking completed with warnings"` which means it never fails anyway -- tests are the real gate.

    **Do NOT change:**
    - The workflow triggers (push/PR branches)
    - The deploy jobs (deploy-preview, deploy-staging, deploy-production)
    - The notify-failure job
    - The node version or checkout steps

    The `lint-and-test` job steps should be ordered:
    1. Checkout repository
    2. Setup Node.js
    3. Install dependencies
    4. **Run tests with coverage** (NEW)
    5. **Upload coverage report** (NEW)
    6. Run type checking
    7. Build application

    This ordering means: if tests fail, the job fails immediately. Type-checking and build don't run. Since deploy jobs `need: lint-and-test`, they also won't run. This is the gate behavior required by TST-06.
  </action>

Optionally run `npx vitest run --coverage` locally to verify it produces a coverage/ directory.</verify>
