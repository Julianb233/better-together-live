<task type="auto">
  <name>Task 1: Refactor requireAuth to read env from request context</name>
  <files>src/lib/supabase/middleware.ts</files>
  <action>
  Refactor `requireAuth` so it no longer takes `env: SupabaseEnv` as a parameter. Instead, it should read env from `c.env` at request time. This is required because Hono on Vercel provides env per-request, not at middleware registration time.

  Change the signature from:
  ```typescript
  export function requireAuth(env: SupabaseEnv) {
    return async (c: Context, next: Next) => { ... }
  }
  ```
  To:
  ```typescript
  export function requireAuth() {
    return async (c: Context, next: Next) => {
      const env = {
        SUPABASE_URL: c.env?.SUPABASE_URL || '',
        SUPABASE_ANON_KEY: c.env?.SUPABASE_ANON_KEY || '',
        SUPABASE_SERVICE_ROLE_KEY: c.env?.SUPABASE_SERVICE_ROLE_KEY
      }
      const supabase = createClientWithContext(c, env)
      const { data: { user }, error } = await supabase.auth.getUser()
      if (error || !user) {
        return c.json({ error: 'Unauthorized' }, 401)
      }
      c.set('userId', user.id)
      c.set('userEmail', user.email)
      c.set('user', user)
      c.set('supabase', supabase)
      await next()
    }
  }
  ```

  Apply the same refactoring to `optionalAuth` -- remove the `env` parameter, read from `c.env` instead.

  Also add a new `requireAdmin` middleware function:
  ```typescript
  export function requireAdmin() {
    return async (c: Context, next: Next) => {
      const user = c.get('user')
      if (!user) return c.json({ error: 'Unauthorized' }, 401)
      const isAdmin = user.app_metadata?.role === 'admin'
      if (!isAdmin) return c.json({ error: 'Forbidden' }, 403)
      await next()
    }
  }
  ```

  Keep all existing helper functions (getSupabaseFromContext, getUserFromContext, getUserIdFromContext) unchanged.

  IMPORTANT: Search the entire codebase for any existing calls to `requireAuth(env)` or `optionalAuth(env)` and update them to `requireAuth()` / `optionalAuth()` (no args). Use grep to find all usages before and after.
  </action>
  <verify>
  Run `grep -rn "requireAuth\|optionalAuth" src/` to confirm:
  1. The middleware.ts definitions no longer accept env parameter
  2. All call sites use the no-arg form
  3. No TypeScript errors: `npx tsc --noEmit` (or at minimum the middleware file compiles)
  </verify>
  <done>requireAuth() and optionalAuth() read env from c.env at request time. requireAdmin() exists. All call sites updated.</done>

  In src/index.tsx, apply the refactored `requireAuth` globally to all /api/* routes using Hono's `except` combinator from `hono/combine`.

  1. Add import at top of file:
  ```typescript
  import { except } from 'hono/combine'
  import { requireAuth } from './lib/supabase/middleware'
  ```

  2. Replace the existing `app.use('/api/*', cors())` section with the security middleware stack. Place it AFTER the cors middleware but BEFORE any route definitions:
  ```typescript
  // CORS (configured separately in plan 01-04)
  app.use('/api/*', cors())

  // Global auth -- all /api/* routes require auth EXCEPT these public routes
  app.use('/api/*', except(
    [
      '/api/auth/*',
      '/api/auth/supabase/*',
      '/api/payments/webhook',
      '/api/health',
    ],
    requireAuth()
  ))
  ```

  3. Do NOT remove any existing route definitions or API module mounts. Only add the global middleware.

  4. If any individual route handlers currently call `requireAuth(env)` inline (e.g., `someApi.use('/*', requireAuth(env))`), leave them for now -- they will be redundant but not harmful since the middleware was refactored to be idempotent. They can be cleaned up in Phase 2.
