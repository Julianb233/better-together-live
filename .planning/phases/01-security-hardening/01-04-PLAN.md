  <files>src/index.tsx, vercel.json, src/api/auth.ts</files>
  <action>
  **CORS -- Fix both locations (dual CORS problem from research):**

  1. In `src/index.tsx`, replace `app.use('/api/*', cors())` with restricted CORS:
  ```typescript
  app.use('/api/*', cors({
    origin: (origin, c) => {
      const allowed = [
        'https://bettertogether.app',
        'https://www.bettertogether.app',
      ]
      // Allow Vercel preview deployments in non-production
      if (c.env?.NODE_ENV !== 'production' || !c.env?.NODE_ENV) {
        allowed.push('http://localhost:3000', 'http://localhost:5173')
      }
      // Also check CORS_ORIGINS env var for additional allowed origins
      const extraOrigins = c.env?.CORS_ORIGINS?.split(',') || []
      allowed.push(...extraOrigins)
      return allowed.includes(origin) ? origin : ''
    },
    allowMethods: ['GET', 'POST', 'PUT', 'DELETE', 'OPTIONS'],
    allowHeaders: ['Content-Type', 'Authorization'],
    credentials: true,
    maxAge: 86400
  }))
  ```

  NOTE: The `origin` callback in Hono's cors middleware receives `(origin, c)` where `c` is the Context. Use `c.env` to access environment variables. Check the Hono cors docs to confirm the exact callback signature -- if it only takes `origin` (no context), then read CORS_ORIGINS from `process.env` instead.

  2. In `vercel.json`, REMOVE the entire `headers` array. CORS is now handled entirely by Hono middleware. The file should become:
  ```json
  {
    "buildCommand": "npm run build",
    "outputDirectory": ".vercel/output",
    "framework": null
  }
  ```

  **Hardcoded JWT secret -- Remove from auth.ts:**

  3. In `src/api/auth.ts`, find the `getJwtSecret` function (line ~28-30):
  ```typescript
  function getJwtSecret(env: any): Uint8Array {
    const secret = env?.JWT_SECRET || 'better-together-secret-key-change-in-production'
    return new TextEncoder().encode(secret)
  }
  ```

  Replace with:
  ```typescript
  function getJwtSecret(env: any): Uint8Array {
    const secret = env?.JWT_SECRET
    if (!secret) {
      throw new Error('JWT_SECRET environment variable is required')
    }
    return new TextEncoder().encode(secret)
  }
  ```

  This removes the hardcoded fallback. If JWT_SECRET is not set, the app throws an error instead of silently using a known secret.
  </action>

  3. `grep -n "Access-Control-Allow-Origin" vercel.json` returns NO results (CORS removed from vercel.json)
  4. `grep -n "origin" src/index.tsx` shows the restricted CORS config
  </verify>
  <done>CORS restricted to production domains. Hardcoded JWT secret removed. vercel.json CORS headers removed.</done>
