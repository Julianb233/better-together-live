<task type="auto">
  <name>Task 1: Migrate checkins.ts, dates.ts, experiences.ts, intimacy.ts</name>
  <files>src/api/checkins.ts, src/api/dates.ts, src/api/experiences.ts, src/api/intimacy.ts</files>
  <action>
For each of the 4 files, apply this migration pattern:

1. **Replace imports**: Remove `import { createDatabase } from '../db'`. Add `import { createAdminClient, type SupabaseEnv } from '../lib/supabase'`
2. **Replace DB instantiation**: Replace `const db = createDatabase(c.env as Env)` with `const supabase = createAdminClient(c.env as SupabaseEnv)`
3. **Convert queries**:
   - `db.all<any>(sql, params)` -> `supabase.from('table').select('columns').eq('col', val)` etc.
   - `db.first<any>(sql, params)` -> same but with `.single()` or `.maybeSingle()`
   - `db.run(sql, params)` for INSERT -> `supabase.from('table').insert({...}).select().single()`
   - `db.run(sql, params)` for UPDATE -> `supabase.from('table').update({...}).eq('id', id)`
   - `db.run(sql, params)` for DELETE -> `supabase.from('table').delete().eq('id', id)`
4. **Add Zod validation**: Import schemas from `src/lib/validation/schemas/`. Add `zValidator('json', schema, zodErrorHandler)` middleware to POST/PUT/PATCH routes. Add `zValidator('query', querySchema, zodErrorHandler)` to GET routes with query params.
5. **Error handling**: Use `.maybeSingle()` instead of `.single()` for lookups that may return no rows. Check `if (error) throw error` after each Supabase call. Handle PGRST116 where `.single()` is used.

CRITICAL: Do NOT change the response shape or status codes. Keep the same JSON keys the frontend expects. Only change the database layer and add input validation.

CRITICAL: Use `createAdminClient` (service role key, bypasses RLS) for ALL routes during migration -- this matches the current Neon behavior which has no RLS.
  </action>
  <verify>`grep -r "createDatabase\|from.*'../db'" src/api/checkins.ts src/api/dates.ts src/api/experiences.ts src/api/intimacy.ts` returns no results. `grep "createAdminClient" src/api/checkins.ts src/api/dates.ts src/api/experiences.ts src/api/intimacy.ts` returns 4 matches.</verify>
