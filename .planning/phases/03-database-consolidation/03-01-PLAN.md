Read each API route file in `src/api/` to understand what fields each endpoint accepts (request body, query params, URL params). Then create one schema file per domain.

For each schema file, define named exports for each endpoint's input shape. Follow this pattern:

**checkins.ts**: createCheckinSchema (relationship_id: uuid, user_id: uuid, connection_score: 1-10 optional, mood_score: 1-10 optional, gratitude_note: string max 500 optional, support_needed: string max 500 optional, highlight_of_day: string max 500 optional, relationship_satisfaction: 1-10 optional)

**goals.ts**: createGoalSchema, updateGoalSchema (with .partial())

**activities.ts**: createActivitySchema, activityQuerySchema (with category, type filters)

**users.ts**: updateProfileSchema, userQuerySchema

**discovery.ts**: searchQuerySchema (q: string min 2, type: enum all/users/communities/posts, page, limit), discoverQuerySchema

**messaging.ts**: sendMessageSchema (conversation_id: uuid, content: string 1-2000), createConversationSchema

**communities.ts**: createCommunitySchema, createPostSchema, communityQuerySchema

**relationships.ts**: createRelationshipSchema, invitePartnerSchema

**payments.ts**: createCheckoutSchema, webhookSchema

**social.ts**: followSchema, blockSchema, reportSchema, createPostSchema

Read each actual API file to get the exact field names and types. Do NOT guess -- use what the code currently destructures from `c.req.json()` or `c.req.query()`.

Import common schemas from `./common.ts` where appropriate (pagination, uuid, scores).
